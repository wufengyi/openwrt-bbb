Get one byte of input from the serial driver.

This function is to be called from the actual R\+S232 driver to get one byte of serial data input.

For systems using low-\/power C\+P\+U modes, the return value of the function can be used to determine if the C\+P\+U should be woken up or not. If the function returns non-\/zero, the C\+P\+U should be powered up. If the function returns zero, the C\+P\+U can continue to be powered down.
